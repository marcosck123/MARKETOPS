# Claude Project — Design, UX e UI Profissional

Este pacote foi criado para configurar um Claude Project/Chat especializado em **Design, UX, UI, Produto, Design System, Acessibilidade e prompts para Codex**.

A proposta é fazer o Claude trabalhar como uma equipe de produto/design, com análise profunda, mas mantendo o resultado visual **minimalista, simples, limpo e funcional**.

## Como usar no Claude

1. Crie um novo Project no Claude.
2. Nome sugerido: `Design UX UI — Equipe de Produto`.
3. Em **Project Instructions**, cole o conteúdo do arquivo:
   - `00_PROJECT_INSTRUCTIONS.md`
4. Em **Knowledge / arquivos do projeto**, envie todos os outros arquivos `.md`.
5. Sempre que iniciar um novo chat dentro desse Project, peça:

```txt
Leia as instruções do projeto e use os arquivos de conhecimento como base. Antes de criar qualquer tela, valide fluxo, botões, redirecionamentos, estados, módulos e riscos de UX/UI.
```

## Arquivos

- `00_PROJECT_INSTRUCTIONS.md` — instruções principais para colar no Project.
- `01_MASTER_DESIGN_UX_UI_SYSTEM.md` — manual completo da equipe de design.
- `02_UX_FLOW_AND_NAVIGATION_CHECKLIST.md` — checklist de fluxo, botões, CTAs e redirecionamentos.
- `03_DESIGN_SYSTEM_AND_VISUAL_STYLE.md` — regras de estilo visual, componentes e consistência.
- `04_SCREEN_ANALYSIS_FRAMEWORK.md` — modelo de análise profunda para qualquer tela.
- `05_ACCESSIBILITY_QA_AND_INTERFACE_STATES.md` — acessibilidade, QA e estados obrigatórios.
- `06_CODEX_PROMPT_FACTORY.md` — como transformar análise de design em prompt para Codex.
- `07_PROJECT_DECISIONS_TEMPLATE.md` — arquivo para registrar decisões do projeto.
- `08_SCREEN_BRIEF_TEMPLATE.md` — template para você preencher antes de pedir uma tela.

## Princípio central

A análise pode ser pesada. O visual deve continuar simples.

Design bom não é o mais chamativo. Design bom é o que o usuário entende rápido, usa sem esforço e conclui a tarefa sem confusão.

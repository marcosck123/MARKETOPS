## Codebase Patterns

- node_modules está em Docker volume (marketops_node_modules). npm install só funciona dentro do container.
- Checks (typecheck, lint, build) requerem Docker rodando. Rodar com: `docker compose run --rm app npx tsc --noEmit` e `docker compose run --rm --no-deps app npm run lint`.
- Após US-001: rodar `docker compose run --rm app npx prisma migrate dev --name init` e depois `docker compose run --rm app npx prisma db seed` para aplicar schema e dados.
- IDs do schema Prisma são strings para compatibilidade com mock data existente (ex: "alimentos", "arroz-feijao").
- Singleton Prisma client em src/lib/db.ts — padrão obrigatório para Next.js dev mode.
- DATABASE_URL usa hostname "db" (resolve apenas dentro da Docker network).
